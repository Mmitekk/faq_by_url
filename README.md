# FAQ by URL

Модуль для Drupal 9.5+ / 10+ / 11, позволяющий добавлять вопросы-ответы (FAQ) и привязывать их к страницам сайта по URL. Вывод осуществляется через блок, размещаемый в Схеме блоков.

## Возможности

- **Привязка по URL** — вопрос-ответ привязывается к конкретным URL страниц (таксономия, ноды, представления), а не к сущностям
- **Множественная привязка** — один вопрос-ответ может отображаться на нескольких страницах
- **Поддержка алиасов** — модуль автоматически определяет как системные пути (`/taxonomy/term/5`), так и псевдонимы (`/catalog/metallocherepitsa`)
- **Wildcard-привязка** — поддержка масок URL вида `/catalog/*` для отображения на всех подстраницах
- **Аккордеон** — стильный блок с плавным раскрытием/скрытием ответов
- **SEO** — автоматически добавляется Schema.org JSON-LD разметка (FAQPage)
- **Совместимость** — Drupal 9.5.11, Drupal 10, Drupal 11
- **Кэширование** — корректная интеграция с системой кэша Drupal

## Установка

1. Скопируйте папку `faq_by_url` в директорию `web/modules/custom/` вашего Drupal-сайта
2. Включите модуль через админку (`/admin/modules`) или Drush:
   ```bash
   drush en faq_by_url -y
   ```
3. Модуль автоматически создаст таблицу в базе данных и настройки по умолчанию

## Настройка

### Добавление вопросов-ответов

1. Перейдите в **Конфигурация → Контент → Вопросы-ответы (FAQ by URL)** (`/admin/config/content/faq-by-url`)
2. Нажмите **«Добавить вопрос-ответ»**
3. Заполните форму:
   - **Вопрос** — текст вопроса
   - **Ответ** — текст ответа (поддерживает HTML-форматирование)
   - **URL страниц** — добавьте один или несколько внутренних путей сайта (каждый URL — отдельное поле, нажимайте «Добавить ещё»)
   - **Вес** — порядок сортировки (меньше = выше)
   - **Опубликован** — чекбокс для управления видимостью

### Примеры URL для привязки

| URL | Описание |
|-----|----------|
| `/catalog/metallocherepitsa` | Категория каталога (алиас) |
| `/taxonomy/term/5` | Термин таксономии (системный путь) |
| `/node/42` | Материал (нода) |
| `/contacts` | Страница контактов |
| `/catalog/*` | Все подстраницы каталога (wildcard) |

### Размещение блока

1. Перейдите в **Структура → Схема блоков** (`/admin/structure/block`)
2. Выберите нужную область темы оформления
3. Нажмите **«Разместить блок»**
4. Найдите блок **«FAQ by URL — Вопросы-ответы»**
5. Настройте видимость блока (по желанию можно ограничить отображение определёнными страницами)
6. Сохраните настройки блока

### Настройки модуля

Перейдите в **Конфигурация → Контент → Вопросы-ответы → Настройки** (`/admin/config/content/faq-by-url/settings`):

- **Заголовок блока** — текст заголовка над FAQ
- **Сообщение при отсутствии вопросов** — текст, если для текущей страницы нет FAQ (пустое = скрыть блок)
- **Показывать нумерацию** — добавляет порядковые номера к вопросам
- **Раскрывать первый вопрос** — первый вопрос на странице будет раскрыт по умолчанию

## Логика определения URL

Блок определяет текущий URL страницы и ищет FAQ-элементы, привязанные к этому URL. При этом:

1. Определяется системный путь (например, `/taxonomy/term/5`)
2. Определяется псевдоним (например, `/catalog/metallocherepitsa`)
3. Проверяется Request URI (адрес в браузере)
4. Для каждого FAQ проверяется совпадение по всем вариантам URL
5. Поддерживается wildcard-матчинг (`/catalog/*`)

## Кастомизация

### Переопределение шаблона

Скопируйте `templates/faq-by-url-block.html.twig` в папку вашей темы и измените по необходимости.

### Переопределение стилей

Подключите свои стили в `THEME.libraries.yml` или переопределите CSS-классы:

- `.faq-by-url-wrapper` — обёртка блока
- `.faq-by-url-title` — заголовок
- `.faq-by-url-item` — элемент FAQ
- `.faq-by-url-item--open` — раскрытый элемент
- `.faq-by-url-question` — вопрос
- `.faq-by-url-question__text` — текст вопроса
- `.faq-by-url-answer` — ответ
- `.faq-by-url-counter` — нумерация
- `.faq-by-url-icon` — иконка +/-

### Изменение цвета акцента

Основной цвет акцента — `#e95b18` (оранжевый). Для смены достаточно переопределить CSS-переменные или найти и заменить `#e95b18` в `css/faq-block.css`.

## Удаление модуля

1. Отключите модуль через админку или Drush:
   ```bash
   drush pm:uninstall faq_by_url -y
   ```
2. При удалении модуля Drupal автоматически удалит таблицу `faq_item` и конфигурацию
